Prework Study Guide

HTML

  • The head element: contains information about the webpage.
  • The meta element in the head: contains information about the page that is used by the browser and helps with SEO, the user will not see this information.
  • The body element: represents the visible content shown to the user.
  • Semantic elements: indicate role of their content and give structure to webpage. Where possible use semantic elements as it makes the HTML easier to read for developers and users using accessibility software.
  • The main element: is where the main body of the website sits.
  • The h1 element: should only be used once - it defines the title of the webpage.
  • The footer element: is where information such as the author, copyright and sitemap sits.

CSS

  • inline CSS: written into the opening tag of an element, for one-off permanent styling but not very flexible.
  • internal style sheet: embedded stylesheet into the head section of a webpage using 'style' element. can configure ID and class attributes but limited to only one webpage.
  • external style sheet: linked in the head element. Best practise as most lean on code - one CSS file can style multiple webpages. link using 'link' element.
  • margin: adds space outside element.
  • padding: adds space inside element.
  • Do Not Repeat Yourself (DRY) code: use commas for applying styling to multiple selectors at once.

Git

  • touch style.css: creates a css file called style.

  • git checkout -b branch-name: creates a new branch and switches to it.
  • git status: check you have changed to new branch.
  • git add -A: to add the changed files into staging area of Git.
  • git commit -m "add code to HTML file": to commit all files in the working branch to the repo. -m allows a short descriptive message with commit. Once complete Git repo should be up-to-date with local working directory.
  • git pull origin main: To make sure current base/main branch stored locally is up-to-date.
  • git push origin feature/add-html: push up all changes.
